Ubiquitous language란 개발, 설계, 기획, 디자이너 등 프로젝트 구성원 모두 동일하게 사용하는 언어를 말한다

ubiquitous language 왜필요할까?

동일한 프로젝트를 진행하는 조직 혹은 기업에서 구성원끼리 소통하는 경우 오차를 최소화한다

이게 무슨말이냐. 예를 들어 상품, 식품이라는 단어는 식품회사에서
아래의 예시를 동일한 표현으로 사용할 수있다
e.g)
- 우리 식품의 판매량
- 우리 상품의 판매량

 

표현은 비슷하지만 엄연히 다른 단어이며 회의 중 흐름을 한번이라도 놓쳐 정확한 의도를 파악하지 못하는경우
상품의 표현이 식품이 아닌 다른 표현이 됐을 때 혼동할 수 있다
그리고 개발 시에 하나의 프로젝트를 여러 명이 참여할 경우 변수를 선언할 때 문제 가능성이 있다
비용을 표현하는 경우 아래처럼 다르게 표현한다면
e.g)
- cost
- price

비용과 가격은 비슷하지만 다른 모호한 표현이다. 혼동의 여지가 생길수있다
따라서 정확하게 서로 인지할 수 있는 보편어를 사용한다면 문제의 여지를 최소화 할수있다

 

일반 회사도 물론 중요하며 msa architecture처럼 하나의 service를 여러개의 domain으로 쪼갠다면 더더욱중요하다
다른 팀과 혹은 협업사와 다른의도의 용어가 같은 표현으로 쓰이는 등의 문제를 막아야하기때문이다

 

어떻게 정리하면좋을까

notion이나 다른 협업 툴을 이용해서 모두가 볼 수 있는 곳에 보편어를 정리한다
개발, 기획, 설계 등의 사람들이 다같이 정리하며 변수명으로 사용할 영어와 한글 표현
그리고 설명을 덧붙이면 기본적인 보편어 정리는 끝난다

 

아래처럼 이 아니더라도 규칙을 따로 만들어서 꾸준히 update하는것이 중요하다

  • 카테고리 2 - (카테고리) - 단어(영어), 단어(한글), 설명
  • 카테고리 3 - (카테고리) - 단어(영어), 단어(한글), 설명

사업이나 프로젝트가 고도화되기전에 빠르게 시작하는 것이 바람직하다
모두에게 공유하고 지속적으로 서로 사용하는 단어를 맞춰가는 것이 핵심이다
회의를 진행하더라도 기존용어나 새로운 용어를 정확한 의도로 사용해나가자
