암호화란?

보안이 필요한 data를 사용자 혹은 개발자가 식별할수없게 특정한 algorithm을 사용해 문자열로 변경하는 행위를 말합니다

가려져야하는 data가 노출되어 로직혹은 api 를 해독하거나 간단한  공격하는 것을 방지한다

 

양방향과 단방향의 차이

양방향 algorithm은 암호화가 진행된 암호문을 복호화할수있는 algorithm이다

단방향은 algorithm은 복호화가 불가능한 암호문을 만든다

단방향 암호문은 대표적으로 hash가있다

 

양방향에서 대칭키와 비대칭키는 뭘까?

대칭키는 암호화와 복호화를 진행할때 동일한 key를 사용한다

비대칭은 암호화와 복호화에 다른 key를 사용한다

대칭키(비공개키)
암호화, 복호화가 동일한 key를 사용하기때문에 유출되면안된다

대표적: aes

비대칭키(공개키)

암호화할때의 키는 공개키로 사용한다, 속도가 느려서 대량의 data에는 비효율적이다

대표적: rsa

PKCS#7 패딩:

암복호화할때 input이 암호 블럭사이즈의 배수와 맞지않으면 배수에 맞춰 빈공간을 채워주게된다 이 채워주는 방법을 패딩이라고 부른다

PKCS#5와 PKCS#7 의 차이 는 

PKCS#5는 암호블럭 사이즈가 8 바이트에 맞춰져있다 

AA 07 07 07 07 07 07 07 - 1byte data + 7byte padding

AA BB 06 06 06 06 06 06 - 2byte data + 6byte padding

AA BB CC 05 05 05 05 05 - 3byte data + 5byte padding

이런식으로 8byte에 맞춰져있다

aes 알고리즘은 암호화블럭 16byte를 사용할때 
PKCS#7 은 16 byte까지 있는걸 말하고 padding으로 저렇게 16까지 채운다는 의미이다
